# 複数 agent パターン詳細手順

各タスクに specialist agent を割り当て、独立したセッションで真に並列実行する。agent 間の直接連携は前提にせず、メインセッションが結果を集約する。

## B-1. agent の起動準備

```
`spawn_agent`
```

## B-2. タスクの作成と agent の起動

各タスクに対して `update_plan` でタスクを登録し、`spawn_agent` を**単一メッセージ内で並列に起動**する。

```
spawn_agent({
  description: "<タスクの要約>",
  prompt: `あなたは <task-n> です。以下のタスクを実行してください:

<タスクの詳細な指示>


### 結果の送信
完了したら、変更内容、結果、失敗要因があればそれを最終メッセージで返す。形式:

\`\`\`markdown
## Task Results

**タスク:** <タスク名>
**ステータス:** 完了 / 部分完了 / 失敗
**変更ファイル:** (あれば)
- path/to/file1
- path/to/file2

**結果:**
<詳細な結果>
\`\`\`

### タスク完了
変更したファイルと結果を明記して完了する。`
})

// ... 残りのタスクも同様
```

## B-3. 結果の収集

`wait_agent` で全 agent の完了を待機する。返却された結果をメインセッションで統合する。

## B-4. agent の終了

```
close_agent({ id: "<agent-id-1>" })
close_agent({ id: "<agent-id-2>" })
// ... 残りの agent も同様
```

## フォールバック

- `spawn_agent` が失敗した場合 → パターン A (並列 agent) にフォールバック
- 一部の agent が失敗した場合 → 残りの agent の結果で続行
- 全 agent が失敗した場合 → パターン A にフォールバック
