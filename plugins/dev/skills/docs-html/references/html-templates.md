# カテゴリ別テンプレートガイド

`base-template.html` をベースに、選んだカテゴリの「レイアウト」「コンポーネント」「配色アクセント」を採用する。すべて 1 ファイルに収め、外部 CDN・外部画像に依存しない。

## 共通配色ベース

両テーマで以下の意味色 CSS 変数を `:root` と `[data-theme="dark"]` に定義する:

```
--bg, --bg-elevated, --surface, --border, --text, --text-muted,
--accent, --accent-fg, --success, --warning, --danger, --info
```

light 例: `--bg: #faf9f5; --text: #141413; --accent: #d97757;`
dark 例: `--bg: #1a1a18; --text: #f0eee6; --accent: #f29670;`

カテゴリごとに `--accent` のトーンを変えて雰囲気を出す:

| カテゴリ    | light accent       | dark accent | 雰囲気           |
| ----------- | ------------------ | ----------- | ---------------- |
| `plan`      | `#d97757` (clay)   | `#f29670`   | 構造的・落ち着き |
| `review`    | `#5c8edb` (blue)   | `#7fb0f5`   | 解析・冷静       |
| `design`    | `#a06bd0` (violet) | `#c693ee`   | クリエイティブ   |
| `prototype` | `#3fb27a` (green)  | `#5ed299`   | 動き・実験       |
| `report`    | `#3a86b8` (steel)  | `#6fb3dc`   | 事実・信頼       |
| `explainer` | `#788c5d` (olive)  | `#a3bd80`   | 教育・親しみ     |
| `diagram`   | `#c46a47` (umber)  | `#e89072`   | 図解・温度感     |
| `slide`     | `#222` / `#eee`    | 反転        | ミニマル         |
| `editor`    | `#0bb` (teal)      | `#3dd`      | ツール感         |

## レイアウト指針

### plan (仕様 / 計画)

- ヘッダー: 目的・スコープ・前提条件をカード化
- セクション: フェーズごとに見出し → タスクリスト (チェックボックス、依存タグ)
- データフロー / モックアップは中央にインライン SVG
- 末尾: クリティカルパス、未決事項

### review (PR / コードレビュー)

- 左に差分ビュー (`<pre>` で `+`/`-` を緑/赤背景にハイライト)
- 右側に注釈カード (severity: info/warn/critical を色分け)
- 上部にメタ情報 (PR タイトル、ブランチ、ファイル数、差分行数)
- ファイルツリーをサイドバーに

### design (デザインシステム / カタログ)

- 上部にブランドトークン (カラー、フォント、スペーシング) のチップ
- グリッドで各コンポーネントの実物 + 用途 + Do/Don't
- 折りたたみで HTML / 使用例を表示

### prototype (試作 / 調整 UI)

- 中央に大きなプレビュー領域
- 下/横にパラメータコントロール (`<input type="range">`, トグル)
- 「Copy params (JSON)」ボタンを必ず置く
- `input` イベントでプレビューにライブ反映

### report (報告 / レポート)

- 上部にサマリーカード 3-4 枚 (主要 KPI、状況、判断)
- タイムライン or 表で経過
- 影響範囲・原因・対応・残課題のセクション
- インシデントは「重大度」「初動」「恒久対応」を明示

### explainer (解説 / チュートリアル)

- 左サイドに目次 (アンカーリンク、スクロールスパイ可)
- セクション: 概要 → 図 → コードスニペット → gotchas
- 等幅フォントの注釈、行内コード `<code>` を多用
- モバイル時は目次をトップに折りたたみ

### diagram (図 / フローチャート)

- メインは大きなインライン SVG (テーマ変更で stroke/fill を CSS 変数で連動)
- 凡例ボックスを右下
- 必要なら「ズーム/フィット」ボタン
- 補足説明は SVG 下にコンパクトに

### slide (スライドデッキ)

- `section` を 1 ページとし、表示中の 1 枚以外を `display: none`
- ←/→ キー、PageUp/PageDown でナビ
- 右下にページャー (`3 / 12`)
- スペースキーで次へ、F でフルスクリーン

### editor (カスタム編集 UI)

- フォーム + リアルタイムプレビュー
- ドラッグ&ドロップ (`draggable="true"`) の対応
- 「Copy as JSON」「Copy as Markdown」など出力ボタン必須
- バリデーション/警告をインライン表示

## 共通コンポーネント (再利用)

- **ヘッダー**: 左にタイトル、右に [テーマ切替][言語切替] ボタン
- **コードブロック**: `<pre><code>` + 言語タグ + コピー機能
- **表**: zebra ストライプ、sticky ヘッダー
- **コールアウト**: info / tip / warn / danger の 4 種 (左ボーダー色で区別)
- **タブ**: ARIA `role="tablist"` / `role="tab"` / `role="tabpanel"`
- **アコーディオン**: `<details><summary>` を活用

## レスポンシブ閾値

- `<= 720px`: モバイル想定 (1 カラム、サイドバー隠す)
- `<= 1024px`: タブレット想定 (2 カラムに圧縮)
- `> 1024px`: デスクトップ想定 (本来のレイアウト)

## SVG の扱い

- インライン SVG を採用 (テーマ連動のため)
- 配色は `currentColor` または CSS 変数を `style="fill:var(--accent)"` で指定
- 1 つの SVG が縦/横どちらでも崩れないよう `viewBox` + `preserveAspectRatio` を設定

## コードのシンタックス強調

外部ライブラリを使わない簡易ハイライト:

- 必要に応じてキーワード/文字列/コメントを任意のクラス名 (例: `tok-kw` / `tok-str` / `tok-com`) で `<span>` 包みする
- 対応する色をテーマ用 CSS 変数として `:root` と `[data-theme="dark"]` の両方に追加して着色する
- 出力時に自前で軽量パース or ハードコードでよい (見た目重視)
- `base-template.html` には強調用クラスは含めていないため、必要なときだけ追加する
