# カテゴリ分類

HTML ドキュメントは内容に応じて以下のカテゴリのいずれかに分類する。Step 2 のカテゴリ判定で 1 つ選び、`html-templates.md` の該当カテゴリのレイアウト/コンポーネントを採用する。

## 分類表

| ID          | 用途                                                   | 判定キーワード                                           | 主要コンポーネント                                               |
| ----------- | ------------------------------------------------------ | -------------------------------------------------------- | ---------------------------------------------------------------- |
| `plan`      | 仕様書、実装計画、タスク分解、複数案の探索             | 計画 / 仕様 / spec / RFC / 実装方針 / 探索 / mockup grid | チェックリスト、フェーズ表、データフロー図、モックアップグリッド |
| `review`    | コードレビュー、PR 解説、差分の理解                    | PR / レビュー / diff / コード解説 / リファクタリング     | 差分ビュー、行注釈、重要度ラベル、関連ファイルツリー             |
| `design`    | デザインシステム、コンポーネントカタログ、デザイン比較 | デザイン / UI / コンポーネント / トークン / ブランド     | カラーチップ、コンポーネント一覧、Do/Don't                       |
| `prototype` | アニメーション・インタラクション試作、パラメータ調整   | プロトタイプ / アニメ / インタラクション / スライダー    | スライダー、トグル、preview canvas、copy params ボタン           |
| `report`    | 状況報告、週次/月次レポート、インシデント報告          | レポート / 週報 / 月報 / インシデント / KPI / RCA        | サマリーカード、タイムライン、メトリクス、影響範囲表             |
| `explainer` | 技術解説、概念図、チュートリアル                       | 解説 / explainer / how it works / tutorial / 入門        | セクション目次、ステップ図、コードスニペット、gotchas            |
| `diagram`   | フローチャート、SVG 図、アーキ図、シーケンス           | 図 / フロー / シーケンス / アーキテクチャ / ER           | インライン SVG、凡例、ズーム/フィット切替                        |
| `slide`     | スライドデッキ、社内発表用                             | スライド / deck / プレゼン                               | ページャー、キーボードナビ、進捗インジケータ                     |
| `editor`    | 設定/データの編集 UI、トリアージ、prompt tuner         | 編集 / drag and drop / トリアージ / フラグ管理           | フォーム、ドラッグカード、JSON プレビュー、copy ボタン           |

## 判定ルール

1. **主目的を 1 つに絞る** - 「複数の側面を持つ」場合でも、ユーザーが最も時間を割いて読む / 触る部分を主目的とする
2. **想定読者で迷ったら確認** - 例: 同じ「PR 解説」でも自分の理解整理なら `explainer`、レビュアー向けなら `review`
3. **補助要素は組み合わせる** - 主カテゴリのレイアウトに、他カテゴリのコンポーネント (図、コード、表) を組み込んでよい
4. **どれにも当てはまらない場合** - `explainer` を採用 (汎用的に使いやすい)。必要なら候補を提示してユーザーに短く確認してもよい

## 補助要素の組合せ例

- `plan` + `diagram`: 実装計画 + データフロー図
- `report` + `diagram`: インシデント報告 + タイムライン SVG
- `explainer` + `prototype`: 概念説明 + インタラクティブな試遊
- `review` + `diagram`: PR 解説 + 呼び出し関係グラフ
- `design` + `prototype`: デザイントークン + スライダーで調整

## 参考カテゴリの出典
